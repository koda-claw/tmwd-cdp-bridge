# tmwd-cdp-bridge v0.1.3

Patch release focused on making the real browser release gate reliable on
modern Chrome and Edge builds.

## Highlights

- Fixed the real browser smoke harness for Google Chrome 137+.
- The harness now loads the unpacked extension through browser-level DevTools
  `Extensions.loadUnpacked` instead of depending on removed branded Chrome
  `--load-extension` behavior.
- Preserved the fixed extension id:
  `eghifjkffmcmffejmaaeicejpfopplem`.
- Kept a launch-flag fallback for older Chromium-family browsers that do not
  support the DevTools Extensions domain.
- Updated README and maintainer docs so future release validation uses the new
  loading path.

## Upgrade Notes

- CLI version is now `0.1.3`.
- Extension files did not change; the extension manifest version remains `2.3`.
- Existing installed extensions do not need to be reloaded for this release.
- Agents using the installer or `tmwd-cdp-bridge upgrade` should receive the
  new CLI binary for their platform.

## Validation

- `cargo fmt --all --check`
- `cargo test`
- `node --check` for extension and harness scripts
- `python3 scripts/validate_skill.py skills/tmwd-cdp-bridge`
- Real Microsoft Edge smoke passed with health version `0.1.3`.
- Real Google Chrome smoke passed with `Extensions.loadUnpacked`, fixed
  extension id, normal JS execution, direct CDP, CSP fallback, tab close, and
  extension reload/reconnect.

## Assets

Release archives are published for:

- macOS arm64
- macOS x64
- Linux x64
- Windows x64

Each archive includes the binary, README files, docs, extension, scripts, skill
folder, and a `.sha256` checksum.
